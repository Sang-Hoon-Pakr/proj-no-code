# failures — 테스트 실패 캡처

> 테스트가 빨강이 되면 **즉시** 여기에 기록한다. 수정보다 캡처가 먼저.
> 이 디렉토리는 하네스가 진화하는 근거가 된다.

## 사용법

1. `_template.md` 복사 → `YYYY-MM-DD-<slug>.md` (예: `2026-05-15-ws-reconnect-loop.md`).
2. 사람이 채운다. **AI가 대신 채우지 않는다** — 이해 누락이 룰 품질을 떨어뜨림.
3. 도출된 룰을 해당 `.claude/<module>.md` 에 추가.
4. 코드 수정 후 테스트 통과해도 이 파일은 **보존**. 히스토리가 가치.

## 보존 정책

- 6개월 후 `archive/` 로 이동.
- 같은 패턴 3회 이상 누적 → 자동화 후보 표시 (린트룰/타입가드/CI check).

## 월간 회고

- 자주 나오는 root cause는?
- 룰만으로 못 막은 건 자동화로 옮길 수 있나?
- 더 이상 안 일어나는 룰은 삭제 가능한가? (`.claude/` 다이어트)

## 안티패턴

- 비슷한 실패를 한 파일에 누적 → 금지. 1 failure = 1 file.
- "수정 완료" 후 파일 삭제 → 금지. 보존하라.
- "지금은 바빠서 나중에" → 캡처는 5분, 미루면 영영 안 함.
